Task Name

CREATE Technical INVENTORY

Task No

ASIT02

Task Description

Complete the inventory of all the Items (combination Organisation & Product) within the Sub-Group project scope

Sub-Tasks

 ASIT02-01
Create Inventory using Inventory form DLSD014 provided or review Inventory data provided by Programme Office

ASIT02-02
Check for missing information

ASIT02-03
Update Database

ASIT02-04
Prepare list of Items to be assessed by the Sub-Group

ASIT02-05
Prepare list of vendor supplied Products within the Sub-Group project scope necessiting vendor contact

ASIT02-06
Prepare list of Items ‘not sure’ to be sent to the Programme Office straight away for confirmation if in scope or not

  • Sub-Group scope definition
  • Inventory Form DLSD014 - empty
  • Inventory Form DLSD026 - example
    •

Tools to be used

  

Deliverables
  • List of all Items to be assessed by the Sub-Group
  • List of Products requiring Vendor contact
  • List of items ‘not sure’
  • Database updated

ASIT03A : ITEM/PRODUCT YEAR 2000 ASSESSMENT (INTERNAL)

 

Task Name

ITEM/PRODUCT YEAR 2000 ASSESSMENT (INTERNAL)

Task No

ASIT03A

Task Description

Perform an assessment for each internal Item/Product (type Application) within the Sub-Group project scope to determine its Year 2000 compliance status

Sub-Tasks
  • ASIT03A-01
    Assess each Internal Item/Product using the Item/Product Assessment Form DLSD015 provided by the Programme Office
  • ASIT03A-02
    Input key results in database
    •

Input from Programme Office
  • Item/Product Assessment Form DLSD015 - empty
  • Item/Product Assessment Form DLSD027 - example
    •

Tools to be used

  

Deliverables
  • Item/Product Assessment Form DLSD015 filled in for each Product
  • Year 2000 status updated in Database for each Item and Product
    - Proven Compliant
    - Claimed Compliant
    - Non Compliant
    - Unknown

ASIT04 : IDENTIFY & ANALYSE ITEM DEPENDENCIES

 

Task Name

IDENTIFY & ANALYSE ITEM DEPENDENCIES

Task No

ASIT04

Task Description

Identify all the dependencies (hardware, software, interfaces, etc) for each Item within the Sub-Group project scope.

Dependants Items may be covered by another Sub-Group.

Sub-Tasks

  • ASIT04-01
    For each Item, identify its dependencies with other Items using the Dependencies Form DLSD019
  • ASIT04-02
    Find out which Sub-Group is covering dependant Items
  • ASIT04-03
    Liaise with Programme Office for all unclear responsibilities
  • ASIT04-04
    Liaise with other Sub-Groups or Programme Office to find out Year 2000 compliance status of dependant Items
  • ASIT04-05
    Assess how the Compliance status of the Dependencies affects the Items covered in your Sub-Group
  • ASIT04-06
    Update Database

Input from Programme Office
  • Sub-Group scope definition
  • Dependencies Form DLSD019 - empty
  • Dependencies Form DLSD028 - example
    •

Tools to be used
  • Database
    •

Deliverables
  • Dependencies report

    •

ASIT05 : IDENTIFY ITEM/PRODUCT PLANNED CHANGES

Task Name

IDENTIFY PRODUCT PLANNED CHANGES

Task No

ASIT05

Task Description

Identify for each Item/Product within the Sub-Group project scope all known planned changes that need to be taken into consideration as part of the Year 2000 plan

Sub-Tasks
  • ASIT05-01
    For each Item/Product within the Sub-Group project scope, list the known planned changes (internal or external origin) which need to be taken into consideration when planning the Year 2000 conversion work using the Item/Product Planned Changes Form DLSD020
  • ASIT05-02
    Assess the impact that those changes may have on your Year 2000 Sub-Group workplan
    •

Input from Programme Office
  • Item/Product Planned Changes Form DLSD020 - empty
  • Item/Product Planned Changes Form DLSD029 - example
    •

Tools to be used

 

Deliverables
  • Completed Item/Product Planned Changed Forms
