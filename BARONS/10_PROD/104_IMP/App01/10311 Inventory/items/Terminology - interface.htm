Definition

In the context of Y2K Certification Process an interface between a "start item" S and an "end item" E is understood as the well defined flow of data between the two which triggers a well defined action on either side.

Therefore for a more practical approach an interface here is defined as the collection of items involved and responisble to provide all action needed to complete the successful datatransfer  between the two ends of the interface.

Example

Let's assume that the Telex Application (C401) has an interface (II07) to the PTT Telex Network (N722). Then the Interface (II07) includes action provided by

  1. the starting item (C401)
  2. the end item (N722 )
  3. all other items between (LU15, LU14,WW16,LU10)

Testing

To prove that an interface is compliant you have to prove that all  items in between (and therefore, by definition, all ther supporting items as well) are compliant as. Therefore for a clock forward test  you need to warp both sides of the interface and all items between simultaniously.

Under the assumption that when  items between are compliant and all of them are warped simultaniously, then item screens for testing may be defined just at the both ends of the interface so that testing between the interface may be skiped.

Drawing

Interfaces, which make up the real complexity of the environment, are not shown in the item map, since the complexity of all interfaces may be confusing.

Interfaces are documented separately for every item where they start from during the Item Analysis process and will show up later in every item's Interface Diagram (which is part of the detailed Item Analysis Report).

When drawing the interface diagram  this will be a good crosschek where you will find out that your data highway is broken and that you have to complete the Item Map by adding additional items.

Tip: The interface's items (the items between two items) you will find naturally when you just follow the interface's data path through the item map. Further the Programme Office's Inventory Database (Web2000) is supporting this kind of "research" once the Inventory is complete.

Item Number

The Item Number of an Interface is IInn and must be unique within the organisation.
